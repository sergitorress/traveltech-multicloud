🚀 PASOS RÁPIDOS
════════════════════════════════════════════════════════════════════

PASO 1: Verifica que funciona en LOCAL (5 minutos)
──────────────────────────────────────────────────

Terminal 1 - FRONTEND:
  cd frontend
  npm install
  npm start
  → Abre: http://localhost:3000

Terminal 2 - FAVORITES:
  cd service-favorites
  npm install
  npm start

Terminal 3 - WISHLIST:
  cd service-wishlist
  npm install
  npm start

Terminal 4 - COMMENTS:
  cd service-comments
  npm install
  npm start

Si todo corre sin errores → ¡Sigue al paso 2!

PASO 2: Desplegar en la nube (30 minutos)
──────────────────────────────────────────

Lee: DEPLOYMENT_FREE.md

Y sigue UNA de estas opciones:
  • Railway (RECOMENDADO - más fácil)
  • Render (también bueno)
  • Fly.io (más avanzado)

❓ PREGUNTAS COMUNES
════════════════════════════════════════════════════════════════════

P: ¿Render es realmente gratis?
R: Sí, 1 app es gratis siempre. Para 4 apps debes pagar.
   Por eso usa Railway (es gratis para varias apps).

P: ¿Tengo que usar tarjeta de crédito?
R: Railway pide pero te da crédito gratuito.
   Render no pide. Fly.io tampoco.

P: ¿Dónde están mis datos?
R: En memoria (se pierden si la app se reinicia).
   Para proyectos serios, usa una base de datos.

P: ¿Puedo cambiar después de desplegar?
R: Sí. Puedes mover entre plataformas.
